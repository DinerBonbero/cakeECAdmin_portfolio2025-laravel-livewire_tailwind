## アプリの説明<br>
レスポンシブデザインにも対応した簡易的な管理者機能付きの模擬ケーキ屋ECサイトです。<br>
### 機能

| ユーザー種別     | 商品閲覧 | 商品購入 | 購入履歴 | 商品登録 | 掲載停止 | 再掲載 | 販売履歴 | 複数検索 | 発送状況切替 |
|------------------|----------|----------|----------|----------|----------|----------|----------|----------------|----------------|
| ゲスト           | ✅       | ❌       | ❌       | ❌       | ❌       | ❌       | ❌       | ❌              | ❌            |
| 一般ユーザー     | ✅       | ✅       | ✅       | ❌       | ❌       | ❌       | ❌       | ❌              | ❌            |
| 管理者           | ✅       | ❌       | ❌       | ✅       | ✅       | ✅      | ✅       | ✅              | ✅            |

商品閲覧：商品一覧画面と商品詳細画面で商品情報の閲覧ができます。<br>
商品購入：一般ユーザーのみ商品詳細画面からカートに商品を入れ、カート画面にて商品の数量変更や削除ができます。ユーザー情報を登録完了のユーザーは商品確認画面で商品を購入できます。<br>
購入履歴：一般ユーザーのみ購入履歴一覧画面で購入情報の閲覧ができます。<br>
商品登録：管理者は商品登録画面で販売商品の登録ができます。<br>
掲載停止(論理削除)：管理者のみ商品詳細画で商品の掲載停止(論理削除)ができます。掲載停止にした商品は一般ユーザーのカート内から削除される為、購入ができなくなります。<br>
商品再掲載：管理者のみ商品詳細画で商品の再掲載ができます。<br>
販売履歴：管理者は販売履歴一覧画面で販売情報の閲覧ができます。<br>
複数検索：管理者は販売履歴一覧画面で絞り込み検索(発送状況・表示する期間)ができます。<br>
発送状況切替：管理者は販売履歴画面にて発送状況(発送済み,未発送)の切り替えができます。<br>
             ※発送状況が未発送の際は視認性向上のため背景が赤色になります。<br><br>

## 動作手順は下記をご覧ください<br>
本プロジェクトでは、わたくし自身はbunとXAMPPのMariaDBを使用して開発しましたが、こちらでは手軽なnpmとSQLiteでの動作を前提とした手順を記載させていただきます。なおNode.jsはインストールしているものとします。

### 稼働方法<br>
ターミナルまたはコマンドプロンプトで以下を実行してください<br>

1．プロジェクトを任意のフォルダーにクローンしてください
```
git clone https://github.com/DinerBonbero/cakeECAdmin_portfolio2025-laravel-livewire_tailwind.git
```

2.作成されたプロジェクトフォルダに移動して下記のコマンドを実行してください。<br>

プロジェクトにPHPの依存関係をインストールしてください
```
composer install
```

プロジェクトにJavaScriptパッケージをインストールしてください
```
npm install
```
<br>

3.「.env.example」を複製して「.env」を作成してください
```
cp .env.example .env
```
4.マイグレーションを明示的に初期化したのち、マイグレーションとシーダの実行をしてください
```
php artisan migrate:fresh --seed
```
※下記のレコードをインサートします。<br>
usersテーブルは管理者1名とお客様2名のレコード<br>
user_infoテーブルはお客様2名のレコード<br>
itemsテーブルは各商品のレコード<br>
ordersテーブルにお客様の注文レコード<br>
order_detailsテーブルに注文レコードに対応した注文詳細のレコード<br>

5.シンボリックリンクを設定してください
```
php artisan storage:link
```

6.暗号化キーを設定してください

```
php artisan key:generate
```

7.開発サーバーを起動してください
```
php artisan serve
```

```
npm run dev
```
※マスタシーダでログインされる際はこちらのメールとパスポートでログインできます<br>

| ユーザー      | メール                       | パスワード      | 
|--------------|------------------------------|----------------|
| 管理者        | ```admin@example.com```           | ```cakeAdmin```       |
| お客様A       | ```customer1@example.com```       | ```customer1```       |
| お客様B       | ```customer2@example.com```      | ```customer2```       |
